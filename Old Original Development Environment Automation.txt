# Development Environment Automation Specification

## Document Purpose

The intent is to create a unified system to setup and manage a home lab development environment containing a plethora of machines, both real and virtual, across a collection of operating systems. This specification serves as the comprehensive guide and requirements document for developing an adaptive, flexible automation solution for provisioning and managing a diverse development infrastructure.

The implementation will be realized through a series of Ansible scripts, meticulously defined by the contents of this document. The ultimate deliverables will include:

1. Ansible playbooks
2. All required configuration files
3. A comprehensive handbook describing system setup, usage, and maintenance

## Bootstrap Script

To facilitate easy onboarding of new machines, a bootstrap script will be developed. This script will:

1. Detect the operating system (Linux or Windows).
2. Install essential tools (e.g., `winget`, `scoop`, Ansible, and Python).
3. Clone the Ansible playbook repository.
4. Run an initial Ansible playbook to complete the setup.

### Example Bootstrap Script

```bash
#!/bin/bash

# Detect platform
if [[ "$OSTYPE" == "linux-gnu"* ]]; then
    echo "Linux detected."
    sudo apt update
    sudo apt install -y ansible git python3 python3-pip
elif [[ "$OSTYPE" == "win"* ]]; then
    echo "Windows detected."
    powershell.exe -Command "& {Start-Process winget install --source winget.git -Wait}"
else
    echo "Unsupported OS: $OSTYPE"
    exit 1
fi

# Clone the Ansible repository
git clone https://github.com/<your-repo>.git ~/ansible-playbooks

# Run the bootstrap playbook
cd ~/ansible-playbooks
ansible-playbook -i inventory/hosts bootstrap.yml
```

## Static Handbook Website

The handbook will be a self-hosted static website, written in Markdown, converted to HTML, and served locally using a simple web server.

### Hosting Plan

1. **Markdown to HTML Conversion**:

   * Use tools like `mkdocs` or `Hugo` for generating the static site.

2. **Bundled Web Server**:

   * Use Python's built-in HTTP server for simplicity.

3. **Example Web Server Script**:

```bash
#!/bin/bash
python3 -m http.server --directory ./handbook 8000
echo "Handbook available at http://localhost:8000"
```
